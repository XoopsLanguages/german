Hilfe: Vorlagen

Beschreibung

XOOPS ermöglicht eine gewisse Trennung zwischen der eigentlichen Präsentation der Website (dem „Skin“ oder der Benutzeroberfläche) und dem Code, der im Hintergrund läuft. Ersteres wird durch einen ausgeklügelten Mechanismus basierend auf grafischen Themes gesteuert. Ein Theme kann in seiner Grundform die wesentlichen HTML-Vorlagen enthalten, die die visuelle Benutzeroberfläche der Website definieren, sowie die benötigten Stylesheets und Bilder, um diese visuelle Oberfläche zu ergänzen. Themes können in der Administrationsoberfläche gewechselt werden, sodass das gesamte Erscheinungsbild einer XOOPS-Website nur durch das Ändern eines Auswahlwerts und ein Klick auf „Akzeptieren“ verändert werden kann. XOOPS bietet Website-Administratoren auch die Möglichkeit, Benutzern die Auswahl aus mehreren verfügbaren Themes zu erlauben.

Allgemein gesprochen ist ein Theme der übergeordnete visuelle Rahmen einer Website: Das Theme definiert beispielsweise das allgemeine Layout der Website; es definiert jedoch in der Regel nicht die spezifische Verteilung von Inhalten innerhalb eines Blocks. Dies wird von einer anderen Entität namens Vorlagensatz übernommen. Ein Vorlagensatz ist ein Ensemble von HTML-Vorlagen, die die Präsentation von Inhalten in allen Blöcken einer XOOPS-Website definieren. Um dies mit einer klareren Metapher zu beschreiben, könnte man sagen, dass das Theme der Architekt eines Hauses ist, während der Vorlagensatz der Innenarchitekt ist. Der erste setzt die Wände und definiert die allgemeine Verteilung, aber der zweite ist für das „Ankleiden“ dieser Wände zuständig.

Wenn das aktive Theme ein einfaches ist, das nur eine Themendatei, CSS-Dateien und einige Bilder enthält, verwendet XOOPS den Standard-Vorlagensatz, um die Präsentation von Blöcken zu definieren. Einige komplexe Themes sind jedoch so ausgeklügelt, dass sie ihren eigenen Vorlagensatz enthalten. In diesem Fall muss der Website-Administrator die Administrationsoberfläche verwenden, um sowohl das Theme als auch den Vorlagensatz festzulegen.

Wie wird die Magie des Themes erreicht? Das heißt, wie können HTML-Vorlagen dynamische Inhalte enthalten? Dies geschieht nicht mit Spiegeln und Rauch, sondern mit einer cleveren Vorlagen-Engine namens Smarty, die die Funktion der sogenannten Variableninterpolation übernimmt. Einfach ausgedrückt, verwendet das XOOPS-System PHP-Dateien, um die Logik der Module zu speichern (Zugriff auf die Datenbank, Zuweisung von Variablen, Verwaltung von Rechten...), HTML-Dateien mit eingefügten Smarty-Variablen, um die eigentliche visuelle Oberfläche zu verwalten, und die Smarty-Engine, um Werte und Zeichenfolgen an bestimmten Stellen innerhalb des HTML-Codes einzufügen, der die vom Endbenutzer gesehenen Seiten erstellt. Zusammengefasst bedeutet dies für Sie: Dank seines Themensystems ermöglicht XOOPS Website-Designern, interessantere und visuell beeindruckendere Websites zu erstellen, ohne sich allzu sehr um die Abläufe im Hintergrund kümmern zu müssen.

Vorlagensätze klonen und herunterladen

Wenn Sie einen benutzerdefinierten Vorlagensatz erstellen möchten, können Sie eine Kopie des Standard-Satzes (oder eines anderen) erstellen, indem Sie den Link „klonen“ auswählen (Sie werden aufgefordert, dem neuen Satz einen Namen zu geben). Im Gegensatz zum Standard können andere Vorlagensätze frei bearbeitet werden. Sie können auch eine Kopie eines Vorlagensatzes auf die Festplatte speichern, indem Sie den Link „herunterladen“ auswählen, oder ihn mit dem Link „löschen“ entfernen. Gelegentlich sehen Sie möglicherweise auch einen zusätzlichen Link „Generieren“ neben neuen Modulen, für die noch keine Vorlagen erstellt wurden.

Vorlagen bearbeiten

Sie können eine vollständige Liste der verfügbaren Vorlagen anzeigen, indem Sie die Verzeichnisbaumansicht durchsuchen. Wenn Sie eine Vorlage auswählen, öffnet XOOPS sie in einem Editor, in dem Sie sie bearbeiten können.

Beim Bearbeiten von Vorlagen müssen Sie eine Einstellung aktivieren, die Modulvorlagen-.tpl-Dateien automatisch aktualisiert, wenn es neuere Dateien im Verzeichnis themes/Ihr-Theme/templates für das aktuelle Theme gibt (Systemmodul -> Einstellungen -> Allgemeine Einstellungen -> Modulvorlagen-.tpl-Dateien aus dem Verzeichnis themes/Ihr-Theme/templates aktualisieren? -> „Ja“). Diese Einstellung sollte deaktiviert werden, sobald die Website öffentlich zugänglich ist.

Wenn Sie versehentlich eine Vorlage beschädigen, denken Sie daran, dass Sie eine „funktionierende“ Kopie aus dem Standard-Vorlagensatz abrufen können. Wenn Sie mit nicht-standardmäßigen Vorlagen arbeiten, ist es möglicherweise eine gute Idee, vor Beginn der Bearbeitung eine Kopie auf die Festplatte zu speichern.
